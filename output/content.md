## SLIDE 4: 핵심 개념 — 에이전트 팀의 3가지 구성 요소
- 레이아웃: 중앙 허브-스포크 다이어그램
- 제목: 에이전트 팀을 구성하는 3가지 핵심 요소
- 핵심 내용:
  - [중앙 허브] 오케스트레이터 (Team Lead / 메인 Claude)
  - [스포크 1] Sub-agents — 독립 컨텍스트에서 특화 작업 실행 (최대 7개 병렬)
  - [스포크 2] Skills — 재사용 가능한 워크플로우 모듈 (/batch, /loop, /simplify 등)
  - [스포크 3] Hooks — 이벤트 기반 자동 품질 게이트 (파일 저장 → 자동 Lint 등)
- 시각 요소: 3색으로 구분된 허브-스포크 다이어그램, 각 요소에 아이콘 부여
- 핵심 메시지 박스: "Sub-agents는 실행자, Skills는 레시피, Hooks는 규칙"

**발표자 노트:**
이 슬라이드는 전체 발표의 목차 역할을 합니다. 세 가지를 명확히 구분해서 설명하세요. "요리 비유"를 사용하면 효과적입니다 — Skills는 레시피북, Sub-agents는 요리사, Hooks는 위생 검사관. 이후 슬라이드에서 각각을 자세히 설명한다고 예고하세요.

## SLIDE 5: Sub-agents — 나만의 전문가 팀을 만든다
- 레이아웃: 상단 개념 설명 + 하단 코드 예시 카드
- 제목: Sub-agent — 역할과 권한이 분리된 특화 AI 어시스턴트
- 핵심 내용:
  - Sub-agent = 독립 컨텍스트 + 커스텀 역할 + 특정 도구 권한
  - 메인 대화를 오염시키지 않고 병렬로 최대 7개 동시 실행
  - 파일 하나로 생성: YAML 헤더 + 역할 지시문
- 코드 예시 박스:
  ```yaml
  name: code-reviewer
  description: 코드 품질과 보안을 검토하는 전문가
  tools: Read, Grep, Glob, Bash
  model: sonnet
  ```
- 빌트인 Sub-agent 목록 (간결하게): Explore(탐색), Plan(계획), General-purpose(실행), Bash(터미널)
- 시각 요소: "1명의 개발자 → 7개 병렬 에이전트" 비교 일러스트

**발표자 노트:**
"파일 하나만 만들면 전문가 팀원이 생깁니다"라는 말로 시작하세요. 기술적 복잡성보다 결과에 초점을 맞추세요. 코드 예시는 "이 정도면 우리도 할 수 있겠다"는 자신감을 심어주기 위한 것입니다. 의사결정자에게는 "IT 팀이 반나절이면 구축할 수 있는 수준"이라고 설명하세요.

## SLIDE 6: Skills — 한 번 만들어 계속 쓰는 자동화 레시피
- 레이아웃: 좌우 2분할 (개념 설명 + 실사용 예시)
- 제목: Skills — 반복 업무를 /명령어 하나로 처리하는 모듈
- 핵심 내용:
  - [좌측] Skills 개념: SKILL.md 파일로 정의, /명령어로 즉시 호출, 팀 전체 공유 가능
  - [우측] 빌트인 Skills 실사용 예시:
    - `/batch` — 전체 코드베이스를 30개 작업으로 분해해 병렬 마이그레이션
    - `/loop` — 주기적 모니터링 자동화 (배포 감시, PR 알림)
    - `/simplify` — 3개 병렬 리뷰 에이전트로 코드 품질 자동 검토
- 시각 요소: Skills 디렉토리 구조 트리, "한 번 작성 → 팀 전체 재사용" 화살표
- 강조 박스: "사내 브랜드 PPT 템플릿을 Skill로 등록 → 매번 지시 불필요"

**발표자 노트:**
Skills의 진정한 가치는 "팀 지식의 코드화"입니다. 시니어 개발자가 알고 있는 코딩 컨벤션, 보고서 형식, 배포 절차를 Skills에 담으면 주니어도 동일한 품질로 실행할 수 있습니다. PPT 사례는 개발자가 아닌 기획자/임원에게도 공감을 주는 예시입니다.

## SLIDE 7: 실제 사례 1 — 코드 리뷰 자동화
- 레이아웃: 스토리텔링 타임라인 (Before → Process → After)
- 제목: 코드 리뷰 3시간 → 15분 — Fountain의 실전 사례
- 핵심 내용:
  - [Before] PR 하나당 수동 리뷰: 보안 검토 + 성능 분석 + 테스트 커버리지 = 평균 3시간
  - [Process] 3개 Sub-agent가 병렬로 동시 검토:
    - Agent 1: 보안 취약점 스캔
    - Agent 2: 성능 병목 분석
    - Agent 3: 테스트 커버리지 확인
  - [After] Fountain(인력 관리 플랫폼): 계층형 멀티에이전트로 전체 코드베이스 리뷰 자동화
- 통계 강조: "엔지니어 1인당 일일 PR 병합 수 +67% 증가" (Anthropic 내부 데이터)
- 시각 요소: 병렬 처리 다이어그램 (3개 에이전트 동시 작동 애니메이션)

**발표자 노트:**
"지금 여러분 팀의 PR 리뷰에 얼마나 시간이 걸리나요?" 청중에게 물어보세요. 67% 향상이라는 숫자는 Anthropic 내부 데이터이므로 신뢰도가 높습니다. Fountain 사례는 규모가 있는 기업이 이미 도입했다는 사실을 증명하는 레퍼런스로 활용하세요.

## SLIDE 8: 실제 사례 2 — 대규모 코드 마이그레이션
- 레이아웃: 중앙 임팩트 수치 + 상세 설명
- 제목: C 컴파일러를 처음부터 만든 16개의 에이전트
- 핵심 내용:
  - Anthropic 연구팀 실험: 16개 에이전트 팀이 Rust 기반 C 컴파일러 처음부터 제작
  - 결과: 10만 줄 규모 컴파일러, Linux 6.9를 x86 / ARM / RISC-V에서 부팅 성공
  - 대부분의 컴파일러 테스트 스위트에서 99% 통과율
  - 소요: 약 2,000회 Claude Code 세션, API 비용 $20,000
  - 실용 사례: `/batch` 명령으로 `src/` 전체를 Solid에서 React로 마이그레이션 (5~30개 독립 단위 병렬 실행)
- 인용구 박스: "16비트 x86 에이전트 팀이 처음부터 C 컴파일러를 구축해 Linux 커널을 컴파일했다." — Nicholas Carlini, Anthropic 연구원
- 시각 요소: 16개 에이전트 네트워크 그래프, 10만 줄 카운터 애니메이션

**발표자 노트:**
이 슬라이드의 목적은 "가능성의 한계"를 보여주는 것입니다. 컴파일러를 만들 필요는 없지만, 이 수준의 복잡도를 처리할 수 있다면 여러분의 업무는 확실히 커버됩니다. $20,000이라는 비용은 "비싸다"가 아니라 "인건비 대비 얼마나 저렴한가"로 리프레이밍하세요. 10만 줄을 사람이 짜면 얼마일까요?

## SLIDE 9: 실제 사례 3 — 문서 및 보고서 자동화
- 레이아웃: 워크플로우 플로우차트
- 제목: 판매 데이터 → Excel → PPT → 이메일 초안 — 앱 전환 없이 자동 완성
- 핵심 내용:
  - 통합 자동화 워크플로우:
    1. 판매 데이터 수집 (SQL/BigQuery 자동 쿼리)
    2. Excel 업데이트 자동화
    3. PPT 슬라이드 자동 갱신 (브랜드 가이드라인 유지)
    4. 이메일 초안 생성 및 발송 준비
  - Anthropic × Microsoft 통합 (2025년 발표): Excel, PowerPoint 공유 컨텍스트
  - 다국어 버전 생성: 레이아웃 유지하면서 텍스트만 자동 번역
  - `claude-office-skills`로 PPTX, DOCX, XLSX 직접 생성 및 편집
- 시각 요소: 4단계 자동화 파이프라인 아이콘 플로우 (Data → Excel → PPT → Email)
- 강조 박스: "주간 보고서 작성 시간 90% 절감 가능"

**발표자 노트:**
이 슬라이드는 개발자가 아닌 기획자, 관리자에게 가장 강하게 어필합니다. "주간 보고서를 Claude Code에 맡기면 어떻게 될까요?" 직접 질문하세요. Microsoft Office 통합은 2025년에 공식 발표된 사항으로, 익숙한 도구와의 연결성을 강조하면 거부감이 낮아집니다.

## SLIDE 11: 에이전트 팀 구성 모범 사례 — 5가지 원칙
- 레이아웃: 넘버드 리스트 + 아이콘
- 제목: 효과를 3배로 만드는 에이전트 팀 구성 5원칙
- 핵심 내용:
  1. **팀 규모 최적화** — 대부분의 워크플로우에서 3~5명의 Sub-agent가 최적. Teammate당 5~6개 태스크
  2. **역할 명확화** — 각 에이전트는 명확히 구분된 파일/모듈을 소유. 보안/성능/테스트 역할 분리
  3. **작업 단위 크기 조정** — 너무 작으면 조율 오버헤드, 너무 크면 낭비. 명확한 결과물을 갖는 독립 단위로
  4. **컨텍스트 공유** — CLAUDE.md로 팀 전체에 프로젝트 가이드라인 배포
  5. **품질 게이트 적용** — Hooks로 자동 품질 검사: 파일 편집 후 자동 Lint, `.env` 수정 차단
- 시각 요소: 5단계 수직 타임라인, 각 원칙에 색상 코딩

**발표자 노트:**
이 슬라이드는 "따라 할 수 있는 레시피"를 제공하는 것이 목적입니다. 각 원칙을 30초씩 간략히 설명하세요. 특히 "품질 게이트"는 "AI가 실수를 해도 자동으로 잡아준다"는 안심 메시지로 활용하세요. 보수적인 의사결정자의 리스크 우려를 이 지점에서 해소할 수 있습니다.
